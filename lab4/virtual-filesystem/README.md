# Virtual FileSystem

Виртуальная файловая система с поддержкой CLI и веб-интерфейса.

## Описание

Программная система моделирует файловую систему с поддержкой:
- Создание файлов и папок (`touch`, `mkdir`)
- Навигация (`cd`, `ls`, `pwd`, `tree`)
- Чтение и запись (`cat`, `write`)
- Копирование, перемещение, удаление (`cp`, `mv`, `rm`)
- Управление правами доступа (`chmod`, `chown`, `lsperm`)
- Архивация и распаковка (`archive`, `extract`)
- Резервное копирование и восстановление (`backup`, `restore`)
- Организация файлов по категориям (`organize`)

### Разделение кода

Общий код (модели + операции) используется **и CLI, и веб-интерфейсом**:

- `core/` + `models/` + `operations/` — **общий слой** (domain model)
- `cli/` — интерфейс командной строки
- `web/` — веб-интерфейс (Flask)

Оба интерфейса работают с одним и тем же объектом `FileSystem`.

## Установка

```bash
# Клонировать репозиторий
git clone https://github.com/<username>/virtual-filesystem.git
cd virtual-filesystem

# Установить зависимости
pip install -r requirements.txt
```

## Запуск

### CLI-режим

```bash
python main.py --cli
```

Или просто:

```bash
python main.py
```

### Веб-режим

```bash
python main.py --web
```

Откройте в браузере: [http://127.0.0.1:5000](http://127.0.0.1:5000)

Можно указать порт:

```bash
python main.py --web --port=8080
```

## Веб-интерфейс

Веб-интерфейс предоставляет:

- **Дерево файлов** — навигация по структуре в боковой панели
- **Таблица файлов** — содержимое текущей папки с информацией о размере, правах, владельце
- **Панель инструментов** — кнопки для создания файлов/папок, архивации, бэкапа, организации
- **Контекстные действия** — для каждого файла: просмотр, запись, копирование, перемещение, удаление, изменение прав
- **Индикатор диска** — визуальное отображение занятого/свободного места
- **Журнал операций** — лог всех выполненных действий

| POST | `/api/mkdir` | Создать папку |
| POST | `/api/write` | Записать в файл |
| POST | `/api/cat` | Прочитать файл |
| POST | `/api/cp` | Копировать |
| POST | `/api/mv` | Переместить |
| POST | `/api/rm` | Удалить |
| POST | `/api/chmod` | Изменить права |
| POST | `/api/chown` | Сменить владельца |
| POST | `/api/archive` | Создать архив |
| POST | `/api/extract` | Распаковать архив |
| POST | `/api/backup` | Создать бэкап |

| POST | `/api/organize` | Организовать файлы |

- JSON (хранение состояния)

## Принцип работы

Приложение реализует клиент-серверную архитектуру:

- **Сервер** (Flask) обрабатывает HTTP-запросы и вызывает операции над моделью `FileSystem`
- **Клиент** (браузер) отправляет AJAX-запросы к API и отображает результаты
- Протокол взаимодействия: **HTTP** (GET/POST), формат данных: **JSON**
- Состояние сохраняется в файл `data/state.json` после каждой операции
