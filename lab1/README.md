## Цель работы
Изучить основные возможности языка Python для разработки программных систем с интерфейсом командной строки (CLI). Разработать программную систему, моделирующую файловую систему с возможностью создания, копирования, перемещения, удаления файлов и папок, управления правами доступа, архивации, резервного копирования и организации файлов по категориям.

## Описание предметной области
Программная система эмулирует работу файловой системы на компьютере. Основные сущности:
- **Файл** – хранит содержимое (байты) и имеет формат (расширение).
- **Папка** – контейнер для файлов и других папок.
- **Диск** – логическое устройство с фиксированным объёмом, содержащее корневую папку.
- **Пользователь** – имеет имя, идентификатор, список групп и набор прав.
- **Права доступа** – определяют, может ли пользователь читать, записывать или исполнять файл/папку (реализованы по аналогии с Unix: три октета для владельца, группы и остальных).
- **Форматы файлов** – перечисление поддерживаемых типов (TXT, PDF, JPG, ZIP, PY, JSON, EXE) с возможностью определения по расширению и категоризации.

Все операции с файловой системой проверяют права доступа и генерируют исключения при нарушении.

## Архитектура программной системы

### Основные модули
Проект организован в следующие пакеты:

- **`core/`** – ядро системы. Содержит класс `FileSystem`, управляющий состоянием (пользователи, диски, текущий контекст), загрузкой/сохранением состояния в JSON и разрешением путей.
- **`models/`** – модели данных: `Disk`, `Folder`, `File`, `User`, `Permissions`, `FileFormat`, `FileSystemEntry`.
- **`operations/`** – реализация всех операций над файловой системой (каждая операция – отдельная функция).
- **`cli/`** – интерфейс командной строки. Класс `CLI` читает команды, разбирает аргументы и вызывает соответствующие функции из `operations`.

### Модели данных
- **`FileSystemEntry`** (абстрактный) – базовый класс для файлов и папок. Содержит общие атрибуты: имя, владелец, группа, права, даты создания/изменения, ссылку на родителя. Методы: `get_path()`, `disk` (свойство), абстрактный `get_size()`.
- **`File`** – наследник `FileSystemEntry`. Добавляет атрибуты: формат файла (`FileFormat`) и содержимое (байты). Реализует `get_size()`.
- **`Folder`** – наследник `FileSystemEntry`. Содержит словарь дочерних элементов. Методы: `add_entry()`, `remove_entry()`, `get_entry()`, `list_entries()`, `get_files()` (возвращает только файлы).
- **`Disk`** – содержит ссылку на корневую папку, общий и свободный объём, метку, файловую систему. Методы: `format()`, `update_free_space()`, `resolve_path()`, сериализация.
- **`User`** – представляет пользователя: имя, уникальный ID, дата создания, список групп, словарь дополнительных прав (например, `create_files`).
- **`Permissions`** – хранит три восьмеричных числа (владелец, группа, остальные). Вложенный класс `Access` (IntFlag) определяет биты чтения, записи, выполнения. Методы проверки `can_read/write/execute()`.
- **`FileFormat`** – перечисление (Enum) с элементами, каждый из которых содержит расширение и категорию. Статические методы `from_file()` и `get_folder_name_for_category()`.

### Принципы SOLID
1. **Single Responsibility Principle (SRP)** – каждый класс отвечает за одну чётко определённую часть функциональности:
   - `User` – управление пользователями.
   - `Permissions` – управление правами.
   - `File`, `Folder` – представление элементов файловой системы.
   - `Disk` – управление диском.
   - `FileSystem` – управление глобальным состоянием.
   - `CLI` – взаимодействие с пользователем.
2. **Open/Closed Principle (OCP)** – классы открыты для расширения (например, можно добавить новый тип элемента, унаследовав его от `FileSystemEntry`), но закрыты для модификации существующего кода.
3. **Liskov Substitution Principle (LSP)** – `File` и `Folder` могут использоваться везде, где ожидается `FileSystemEntry`, без нарушения логики (например, метод `get_size()` работает корректно для обоих).
4. **Interface Segregation Principle (ISP)** – интерфейсы не перегружены: `File` не имеет методов для работы с детьми, `Folder` не имеет методов для работы с содержимым файла.
5. **Dependency Inversion Principle (DIP)** – высокоуровневые модули (`CLI`, `FileSystem`) зависят от абстракций (`FileSystemEntry`, а не от конкретных классов). Многие зависимости внедряются через параметры (например, объект `FileSystem` передаётся в CLI).

## Интерфейс командной строки (CLI)

### Команды навигации и просмотра
| Команда | Описание | Пример |
|---------|----------|--------|
| `pwd` | Показать текущий абсолютный путь | `pwd` |
| `ls [путь]` | Показать содержимое папки | `ls`, `ls /home` |
| `cd <путь>` | Сменить текущую папку | `cd docs`, `cd /` |
| `tree [путь]` | Показать дерево папок и файлов | `tree`, `tree /` |

### Команды создания
| Команда | Описание | Пример |
|---------|----------|--------|
| `touch <имя>` | Создать пустой файл | `touch readme.txt` |
| `mkdir <имя>` | Создать папку | `mkdir projects` |

### Команды чтения/записи
| Команда | Описание | Пример |
|---------|----------|--------|
| `write <файл> <текст>` | Записать текст в файл | `write file.txt "Hello"` |
| `cat <файл>` | Вывести содержимое файла | `cat file.txt` |

### Команды копирования/перемещения/удаления
| Команда | Описание | Пример |
|---------|----------|--------|
| `cp <источник> <назначение>` | Копировать файл (только файлы) | `cp a.txt b.txt`, `cp a.txt folder/` |
| `mv <источник> <назначение>` | Переместить/переименовать файл | `mv old.txt new.txt`, `mv file.txt folder/` |
| `rm [-r] <путь>` | Удалить файл или папку (`-r` для папок с содержимым) | `rm file.txt`, `rm -r folder` |

### Команды управления правами доступа
| Команда | Описание | Пример |
|---------|----------|--------|
| `chmod <режим> <путь>` | Изменить права (восьмеричное число) | `chmod 700 secret.txt` |
| `chown <имя_пользователя> <путь>` | Сменить владельца (только admin) | `chown user2 file.txt` |
| `lsperm [путь]` | Показать права доступа элемента | `lsperm`, `lsperm /` |

### Команды архивации
| Команда | Описание | Пример |
|---------|----------|--------|
| `archive <имя_архива> <файл1> [файл2 ...]` | Создать ZIP-архив из файлов | `archive backup.zip a.txt b.txt` |
| `extract <архив> [папка_назначения]` | Распаковать ZIP-архив | `extract backup.zip`, `extract backup.zip /restore` |

### Команды резервного копирования
| Команда | Описание | Пример |
|---------|----------|--------|
| `backup [имя_файла]` | Создать резервную копию текущего диска (JSON) | `backup`, `backup mybackup` |
| `restore <имя_файла>` | Восстановить диск из резервной копии | `restore backup_20250301.json` |

### Команды организации файлов
| Команда | Описание | Пример |
|---------|----------|--------|
| `organize <папка>` | Разложить файлы по подпапкам согласно категориям форматов | `organize downloads` |

### Прочие команды
| Команда | Описание |
|---------|----------|
| `help` | Показать список всех команд |
| `exit` | Выйти из программы (сохраняет состояние) |
